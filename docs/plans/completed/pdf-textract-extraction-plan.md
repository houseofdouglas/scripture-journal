## Decisions & Notes

- Textract Layout prototype was validated against the reference PDF (11-page two-column HBR reprint, `/Users/peter/Downloads/PRM002/Can You Say What Your Strategy Is Article.pdf`) before the spec was written: 118 blocks, median 181 chars, figure junk and running headers dropped by type. The prototype's real Textract output is preserved as PT-04's test fixture at `src/service/__tests__/fixtures/textract-strategy-article.json` (slimmed to PAGE/LAYOUT_*/LINE blocks with geometry stripped, 484 KB — WORD blocks dropped since assembly reads LINE.Text). No need to re-run Textract for tests.
- Presigned-upload design (rather than base64 through Lambda) was chosen during spec review because the user has PDFs over the ~4 MB Function URL payload ceiling. The 50 MB cap is a cost/abuse guard only; cost scales with page count (~$4/1,000 pages), not bytes.
- PT-04 extracts the repeat-across-pages boilerplate filter into a shared util used by both the server path and the retained client (pdf.js) fallback, so the two implementations cannot drift.
- PT-02: skipped adding a scoped S3 IAM statement for `tmp/extract/*` — the existing `lambda_s3` policy already grants `GetObject`/`PutObject`/`HeadObject`/`DeleteObject` bucket-wide (`${bucket_arn}/*`), so a new statement would be redundant dead code. Textract IAM actions (`StartDocumentAnalysis`/`GetDocumentAnalysis`) require `Resource: "*"` — AWS doesn't support resource-level permissions for them, confirmed via the actual `terraform plan` diff. **`terraform apply` was intentionally NOT run** — this task's acceptance criteria only calls for a clean plan, and applying live IAM/Lambda-timeout/S3-CORS/lifecycle changes unattended (mid-autonomous-loop, no user watching) crosses into "affects shared/production infra" territory that warrants a deliberate, confirmed apply step — likely batched with PT-05's deploy once the endpoint code exists to actually exercise it. `terraform plan` was verified clean (3 to add, 1 to change: `lambda_textract` policy, CORS config, lifecycle config, Lambda timeout 30→90) via `cd infra && terraform plan -var="cloudfront_domain=$(terraform output -raw cloudfront_domain)"`.
- PT-03: installing `@aws-sdk/s3-request-presigner` (latest, 3.1080.0) surfaced a type-structural mismatch against the already-installed `@aws-sdk/client-s3@3.1035.0` (`exactOptionalPropertyTypes` flagged mismatched private `Client` fields between SDK versions). Bumped `@aws-sdk/client-s3` to `3.1080.0` to match — no behavior change, just version alignment; full suite re-verified green after the bump.
- PT-03: `getSignedUrl()` signs requests locally and never calls `.send()`, so `aws-sdk-client-mock` (which intercepts at the `.send()` middleware layer) can't mock it — and real credential resolution must never be exercised in unit tests, since `deploy.yml` runs the "Unit tests" CI step *before* "Configure AWS credentials" (zero AWS credentials available at that point). Mocked the `@aws-sdk/s3-request-presigner` module directly in `tmp-upload.test.ts` instead of relying on `aws-sdk-client-mock` for that one function.
- PT-04: reading order is built by walking each Textract `PAGE` block's `CHILD` relationship list (AWS's documented top-to-bottom order) rather than iterating the flat `Blocks` array — this also means a `LAYOUT_LIST`'s nested item blocks are never encountered as top-level page children, so no separate "already consumed" bookkeeping was needed (the plan's task description anticipated needing this; turned out unnecessary given this traversal strategy).
- PT-04: **execution order deviates from the spec's literal FR-4a/4b/4c listing** — the repeat-boilerplate filter (4c) runs *before* continuation-merge (4b), not after. Reasoning, confirmed against the real fixture: a running-header block sitting between two real paragraphs could otherwise get spliced into surrounding text by the merge step before it's recognized as boilerplate and dropped. Filtering first eliminates that risk entirely. Net behavior matches spec intent; only the internal ordering differs from the FR list's sequence, which reads as a set of behaviors rather than a mandated pipeline order.
- PT-04: dropped `filename` from `extractPdf`'s signature (plan/spec sketch had `extractPdf(key, filename)`) — it has no functional use inside the service. `{ filename, pageCount }` logging is a handler-boundary concern per the constitution's layering, so it belongs in PT-05, called with the response's `pageCount` and the request's `filename` at that layer instead.
- PT-04: verified against the *actual* Textract Layout output for the reference PDF (not synthetic fixtures) that: figure/table labels are absent (dropped by type), the recurring "HARVARD BUSINESS REVIEW APRIL 2008" / "PAGE N" footer is entirely absent (dropped by type, not by repeat-count), section headers survive standalone, and hyphenation repair works. One real finding, not a bug: the document's title/byline/opening paragraph is genuinely duplicated in the source PDF across pages 1–2 (a "teaser page then article start" reprint convention) and Textract types the page-2 copy as `LAYOUT_SECTION_HEADER`/`LAYOUT_TEXT` rather than `LAYOUT_HEADER` — so it survives the repeat-filter (only 2 occurrences, below the 5-page threshold for an 11-page document). This is correct: the filter is deliberately conservative (targets true running headers on many pages), and the duplication is real document content, not an extraction artifact — both the old client-side extractor and this one reproduce it identically.
- PT-05/PT-06: PT-06 (backend test completeness gate) was folded into PT-03/04/05 rather than run as a separate pass, same pattern as the archive-articles work. Checked the fold against every Endpoint Behavior and Extraction Quality criterion in the spec; the one gap found — an explicit assertion for the cross-boundary continuation-merge rule and the heading-never-merges rule, which the big fixture exercises implicitly but doesn't pin to a specific expected outcome — was closed with 3 new synthetic-block tests in `pdf-extract.test.ts` (minimal hand-built `Block`/`PAGE` objects, isolated from the fixture's messy real prose). One acceptance criterion is only partially covered: "Textract calls are made in us-east-1" — the bucket/key are asserted, but the hardcoded `region: "us-east-1"` in `textract.ts`'s `TextractClient` constructor isn't itself asserted by a test (would require inspecting client construction args, which `aws-sdk-client-mock` doesn't expose cleanly). Low risk — it's a single hardcoded literal, trivially reviewable — but flagging rather than silently calling it fully covered.
- PT-07: `apiClient` (the shared API wrapper) doesn't support `AbortSignal` — adding it would ripple to every other call site, out of scope here. Implemented the 120s budget as a `Promise.race`-style timeout wrapper around the whole 3-step sequence instead of a true `AbortController` cancellation; this satisfies the spec's "falls back after 120s" requirement functionally, but in-flight requests aren't actually cancelled on timeout (they just get ignored) — a minor inefficiency, not a correctness issue. Wrote `pdf-extract-client.test.ts` now (9 tests covering the full fallback matrix) rather than waiting for PT-09, folding that portion in — only the modal-flow tests remain for PT-09 once PT-08 exists.

Cancel returns to `"url"` mode rather than closing the whole modal, matching the existing "← Back" convention elsewhere in this component — no article is created either way. Title prefill: `suggestedTitle` when Textract found one, else the existing filename-derived default (dashes/underscores → spaces) — unchanged logic, just relocated.

- Post-launch bug (2026-07-09, first real import): on p6 of the reference document, the Wal-Mart sidebar's caption "Source: Jan Rivkin, Harvard Business School" (LAYOUT_TEXT, no terminal punctuation) got continuation-merged with the right-column body continuation "the company offer penny stocks…" — burying a full body paragraph inside a caption, which read as "block truncated + next section never imported." No text was actually lost. Root cause: the merge trigger "lacks terminal punctuation" is too weak. Fix: two-branch trigger — final word starts lowercase, OR block ≥150 chars without terminal punctuation (legit column-ending paragraphs are long; captions/labels/mastheads are short). Verified against the full fixture that exactly 3 bad merges are eliminated (caption, p4 figure text, p1 masthead) and zero legit merges are lost (the p3→p4 "…your strategy: What" continuation is preserved by the length branch — a plain final-word-lowercase rule would have regressed it into a new "truncated block"). Also added block-level dehyphenation on merge (trailing "-" joins without a space). 5 new tests (fixture + synthetic); spec FR-4b updated to match. First-cut lesson recorded: an "ends with lowercase character" check is useless — proper nouns end in lowercase characters ("School"); the signal is the final word's first letter.
- Post-launch bug part 2 (2026-07-09): the merge-refinement above was a partial fix — it stopped the *bad merge* but didn't address the user's actual complaint, which was the sidebar **interleaving** (the Wal-Mart callout physically sits between "…Nor will" and "the company offer penny stocks", so Textract's linear reading order emits main → sidebar → main-continuation, and the continuation reads as orphaned garbage). The complete fix is geometry-based (spec 3a): (1) drop figure/table **labels** (kept text overlapping a figure's box and vertical band — "Source: …", "Customer purchase criteria", "Delivery on criteria" — which Textract types as LAYOUT_TEXT, not LAYOUT_FIGURE, so a type filter alone missed them); (2) relocate **wide** blocks (> 1.4× the column-grid width) — sidebars/callouts spanning multiple columns — plus any heading directly atop them, to the page end, with a merge barrier so main text can't fold into the moved sidebar. This required threading Textract bounding boxes through `orderBlocksByPage` (previously ignored). Two dead-ends recorded before the working version: (a) a full geometric column re-sort *replacing* Textract's reading order made p6 worse (narrow figure-caption fragments formed fake columns and scrambled everything) — the fix is to keep Textract's order and only *remove/relocate* asides, since once the sidebar is out from between them the two body halves are already adjacent in Textract's order; (b) estimating the single-column width per-page skewed on diagram-heavy pages (p4's dozen tiny MISSION/VALUES label blocks dragged the median to 0.182, misclassifying the real 0.295 body columns as "wide" asides and breaking the p3→p4 continuation) — fixed by computing the column width once globally across all pages (median 0.295, stable). Verified across all 11 pages of the reference doc that p6 now reads main-flow-then-sidebar with the continuation re-merged and no caption leakage, and no other page regressed. 4 tests added/updated (fixture assertions + 2 synthetic geometry tests with hand-built bounding boxes); the earlier "keep the caption as a separate paragraph" assertion was superseded (captions are now dropped entirely). Total suite 187 → 194.
