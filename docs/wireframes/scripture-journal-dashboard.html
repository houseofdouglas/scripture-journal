Quick Actions

Annotations — Dashboard (v2)

  1. [1] Browse Scripture — Opens the Scripture Browser (Work → Book → Chapter). Persistent in global nav and sidebar Quick Actions. Both paths lead to the same destination.
  2. [2] Import Article (primary CTA) — Opens the Article Import Modal as an overlay. Styled as the primary action in the nav. Duplicated in sidebar Quick Actions. FR-20.
  3. [3] User menu — Dropdown contains "Change Password" and "Log Out". Shown here as a static overlay for wireframe purposes. FR-05, FR-07.
  4. [4] Multi-entry day group (RESOLVED) — When 2 or more journal entries exist for the same day, they collapse into a single dated group card. Header shows the date + "N entries". Each content piece appears as a compact row: content type · title · note count. No snippet text. Clicking a row navigates to Past Entry View for that specific entry. FR-52, FR-53.
  5. [5] Single-entry day card (RESOLVED) — When only one entry exists for a day, the full card is shown: date block, content type, title, snippet (= text of the user's first saved annotation), and note count. The snippet is annotation text, not source content text. This was a conscious decision — the annotation is more personally meaningful as a memory cue than the scripture text. Could become a user preference in a future phase.
  6. [6] Calendar — Dark-filled cells = days with at least one entry (FR-70, SHOULD). Today's date has an outline. Clicking a filled day filters the entry list to that day (State 3). Empty days are not clickable.
  7. [7] Empty state — Shown when no journal entries exist. Two equal-weight CTAs. Calendar renders but shows no filled days. FR-12 (first-time user flow).
  8. [8] Calendar day selected — When a day with multiple entries is clicked, the grouped card expands (or the list filters to that day). When a day with one entry is clicked, the full card for that day is shown. "← All Entries" returns to the full list.
UX review checklist
  • ✅ Primary action visible in both nav and sidebar
  • ✅ Single-entry days: annotation snippet gives a meaningful memory cue
  • ✅ Multi-entry days: content titles visible at a glance without overwhelming with text
  • ✅ Date grouping is clear — grouped card has a distinct header treatment
  • ✅ Empty state handled
  • ✅ Calendar day selection state shown
  • ⚠️ Open: on a multi-entry grouped day clicked from the calendar, does the entry list expand inline or navigate to a filtered day view? (Current: filtered day view per Flow 5.)
