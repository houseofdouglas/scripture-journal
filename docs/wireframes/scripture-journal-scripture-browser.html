Annotations — Scripture Browser

  1. [1] 2×2 work grid — Four Standard Works presented as equal-weight cards. No hierarchy implied between them. A 2×2 grid fits the exact count and prevents any work appearing more prominent than another. FR-10.
  2. [2] Work card anatomy — Each card shows: a "Standard Works" label (consistent across all four), the work title, and a brief descriptor (testament count, book count, or unique structure note). The → arrow signals that clicking drills down. FR-11.
  3. [3] Breadcrumb — Present at every level below the Work selection. Each crumb is a clickable link back up the hierarchy. "Scripture ›" always returns to the Work grid. Consistent with the Chapter View breadcrumb pattern.
  4. [4] Book list — 3-column grid — Books displayed as compact cards showing title and chapter count. Three columns balances scannability with density. Chapter count helps the user orient themselves before drilling in.
  5. [5] Bible OT/NT grouping — The Bible's 66 books are grouped under "Old Testament" and "New Testament" section headers spanning all columns. Same 3-column grid within each group. The wireframe abbreviates with a "· · · N more books · · ·" row to show the pattern without rendering all 66 entries.
  6. [6] Chapter number grid — 10-column grid of chapter numbers. Dense but scannable — the numbers are the only meaningful label. Clicking any number navigates directly to that Chapter View. FR-11, FR-12.
  7. [7] Chapters with journal entries — Dark-filled chapter tiles indicate the user has at least one journal entry for that chapter, matching the calendar dot convention on the Dashboard. At a glance the user sees where they have been. This is derived from the UserIndex already maintained for the Dashboard.
  8. [8] Chapter legend — Small key below the grid explaining the dark-tile convention. Appears only on the chapter-selection level (Level 3), not on the book or work levels.
  9. [9] D&C sections — skip the book level — D&C has no traditional book subdivisions; sections are the terminal browsable unit. So clicking "Doctrine & Covenants" on the Work grid goes directly to a section number grid (equivalent to Level 3 for other works), skipping Level 2 entirely. Official Declarations are listed separately below the numbered sections as they have a distinct identity. This means D&C is a 2-level path (Work → Section) while other works are 3-level (Work → Book → Chapter).
  10. [10] "Browse Scripture" nav — always returns to Work selection — The nav link is highlighted as active on all three levels of the browser. Clicking it always navigates to /scripture (the Work grid), regardless of current depth. Up-one-level navigation is handled exclusively by the breadcrumb — the nav link is not a substitute for it. This keeps the nav behavior predictable: it's a home button for the scripture browser, not a back button.
  11. [11] Pearl of Great Price — mixed book structure — PoGP contains five books with unequal chapter counts: Moses (8 ch), Abraham (5 ch), JS-History (2 ch), JS-Matthew (1 ch), Articles of Faith (1 ch). The Level 2 book list shows all five using the same card format as BoM. Multi-chapter books display their chapter count; single-chapter books show "1 chapter" and are marked "(opens directly)" to set expectation.
  12. [12] Single-chapter books skip the chapter grid — When a book has exactly one chapter (JS-Matthew, Articles of Faith), clicking the book card navigates directly to the Chapter View at /scripture/pogp/<book>/1. There is no intermediate chapter-number grid to show — a grid of one tile would be meaningless. The "(opens directly)" label on the book card communicates this before the user clicks. FR-11.
UX review checklist
  • ✅ Primary action (select a work) is immediately clear — no secondary actions on this screen
  • ✅ Hierarchy is visually obvious: cards → list → number grid, each level simpler than the last
  • ✅ Breadcrumb present at every sub-level for orientation and back-navigation
  • ✅ D&C's unique structure (no books) handled explicitly rather than with an awkward empty level
  • ✅ Journal history visible at chapter level (dark tiles) without adding clutter at book/work levels
  • ✅ PoGP single-chapter books (JS-Matthew, Articles of Faith) skip the chapter grid and navigate directly to Chapter View — resolved via [12]
  • ✅ "Browse Scripture" nav stays active at all browser levels; clicking always returns to Work selection (/scripture) — resolved via [10]
