The Power of Covenant Belonging
General Conference · April 2024 · Imported Apr 21, 2026 · churchofjesuschrist.org ↗ Today's session
+
Brothers and sisters, I am grateful to be with you this Sabbath morning. As I have reflected on the theme of covenant belonging, I have felt impressed to speak about the sustaining power that comes from understanding our place in God's eternal family.
Each of us entered mortality carrying the imprint of divine parentage. We are not strangers assembled by chance. We are children of a Heavenly Father who knows our names, understands our struggles, and has prepared a path for our return to Him.
The covenant path is not merely a set of obligations. It is a relationship — a living connection between us and our Savior, Jesus Christ. Every ordinance on that path is a declaration that we belong to Him and He to us.
I have seen this belonging manifest in the lives of members across the world — in hospital rooms, in small branches meeting in rented spaces, in family prayer circles, and in quiet moments of personal scripture study. The Spirit confirms belonging when we are faithful to our covenants.

Annotations — Article View

  1. [1] Serif body / sans-serif notes — font family distinction — Article paragraph text renders in Georgia (serif) to signal "source content." Saved annotations and the note editor use the system sans-serif. This satisfies FR-45 and NFR-40 through font family contrast rather than just size or color difference. The distinction is especially important here because both text types appear adjacent within the same paragraph row.
  2. [2] Left gutter with "+" — same pattern as Chapter View — A narrow left gutter (36 px) holds the "+" button, which becomes visible on paragraph hover. Paragraphs that have no "+" visible use an invisible placeholder of the same width so text alignment is consistent across all rows. The "+" does not close the editor (only Cancel does) — consistent with the Chapter View decision. FR-40, FR-41.
  3. [3] Paragraph anchor label in the editor — The editor header shows "Note on paragraph N of M" (1-indexed for display, 0-indexed blockId internally). Articles don't have verse numbers, so this label provides the only positional context. FR-42.
  4. [4] "Today's session" badge in the article header — A small dark badge signals that a live journal entry exists for today. When note count > 0 it updates to "Today's session · N notes." This is the only place note count is surfaced inline on the reading screen. FR-50.
  5. [5] Other-paragraph dimming while editor is open — When an inline editor is open (State 2), non-active paragraphs are dimmed to 45% opacity. This focuses attention on the paragraph being annotated. Clicking a different paragraph's "+" is intentionally not shown — that action is blocked while an editor is open (user must Save or Cancel first). FR-41.
  6. [6] Save error — Retry in the error strip, not the footer — The error message and Retry button appear in a red strip between the textarea and the footer. "Keep Editing" replaces "Cancel" in the footer so the user understands the note text is safe. The primary "Save Note" button is disabled during error state to prevent double-submit confusion. FR-61.
  7. [7] Version notice bar (State 5) — When an article has a previousVersionId, a blue informational bar appears below the nav and above the article header. It states the current version number and import date, and provides a link to the prior version. Annotations made on the prior version are preserved there — stated explicitly to reassure the user nothing was lost. FR-27.
  8. [8] "Version N" badge in article meta — A small pill badge in the article header shows the version number (only when previousVersionId is set). Version 1 articles do not show a badge — the badge only appears to signal that multiple versions exist. FR-27.
  9. [9] Source URL link — opens in new tab — The article header includes a monospaced source URL with ↗ icon to indicate external link. This is informational (the content was fetched and preserved); it does not re-fetch or update the article. FR-25.
  10. [10] Past Entry View — read-only, muted palette — State 7 uses the same layout as the live session but with: (a) muted/desaturated paragraph text, (b) no "+" icons in the gutter, (c) a "Past Entry" banner above the article header stating the session date and a "Study Today →" shortcut, (d) annotations rendered with a lighter left border and desaturated background. The route is /entries/:entryId, not /articles/:articleId. FR-53.
  11. [11] "Study Today →" shortcut in past entry — A button at the top-right of the past entry banner and at the bottom of the content area navigates the user to /articles/:articleId as a fresh today's session. This is equivalent to clicking "Import Article" for an already-imported article — the find-or-create entry logic runs on first save. FR-53, Flow 4.
  12. [12] Back navigation — "← Dashboard" only — Unlike the Scripture Browser / Chapter View (which have a multi-level breadcrumb), Article View has a single flat back link to the Dashboard. Articles do not exist within a browseable hierarchy. Past Entry View uses the same back link but may also have a "← Day's Entries" if entered from the calendar day panel (not shown here).
  13. [13] No prev/next article navigation — Articles are independently imported items, not a linear sequence. There are no "previous article / next article" controls, unlike the chapter prev/next in Chapter View. FR-71 is scripture-specific.
UX review checklist
  • ✅ Primary action (add a note) is discoverable via hover "+" — not buried in a menu
  • ✅ Source content (serif) and notes (sans-serif) are visually distinct font families — FR-45, NFR-40
  • ✅ Save error preserves note text and provides explicit Retry — FR-61
  • ✅ Version badge and notice bar communicate article provenance clearly — FR-27
  • ✅ Past Entry View is read-only with clear visual differentiation from live session
  • ✅ Loading skeleton present so layout shift is minimised on content arrival
  • ✅ Cancel always available in the note editor; modal/editor dismissible without data loss
  • ✅ "Study Today →" shortcut in past entry view reduces friction to start a new session
