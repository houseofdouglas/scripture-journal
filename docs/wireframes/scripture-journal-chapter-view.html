Annotations — Chapter View (v2)

  1. [1] Breadcrumb — Each level is a clickable link back to the Scripture Browser at that level (Work, Book, Chapter list). FR-11. Clicking "Alma" returns to the book's chapter list; clicking "Book of Mormon" returns to the work-level book list.
  2. [2] Prev / Next chapter — Appear at both top and bottom of the chapter (FR-71, SHOULD). "← Alma 31" is disabled on chapter 1 (no previous). "Alma 33 →" is disabled on the last chapter. Shows chapter number out of total to orient the user.
  3. [3] Font legend — Wireframe-only annotation illustrating the font distinction (NFR-40). Does NOT appear in production. Serif = source scripture text; sans-serif = user notes. Different weight/size alone is not sufficient.
  4. [4] Hover state — The "+" button appears in a reserved left gutter (36px wide). Always present but visually hidden until hover, preventing layout shift (NFR-41). Verse row gets a subtle tint to reinforce the affordance.
  5. [5] Verse with saved note — Note appears directly below its verse with a left border bar. Sans-serif font, smaller than verse text, with a "Note" badge and timestamp. FR-43, FR-45.
  6. [6] Add another note — Since annotations are append-only (BR-04), a second thought on the same verse uses this inline link to open a second editor. Notes stack chronologically under the verse.
  7. [7] Active "+" state — RESOLVED — When an editor is open, the "+" turns filled/dark. Clicking it again does nothing (Q1 resolved: only Cancel closes the editor). The filled state is a visual indicator of ownership, not a toggle.
  8. [8] Inline note editor — Anchored below the verse, within the text column. Hint "Append-only · Cannot edit after saving" sets expectation upfront. FR-41, FR-42, BR-04.
  9. [9] Save error — Error inside the editor box, below action buttons. Note text preserved in textarea. Retry re-attempts the save. FR-61.
  10. [10] Loading skeleton — Skeleton bars during initial fetch. Chapter nav disabled until content loads. FR-60.
  11. [11] Navigate-away warning — RESOLVED — When an editor has unsaved text and the user clicks a chapter nav button, an inline warning banner appears immediately below the nav bar (Q2 resolved: warn, don't silently discard). Editor stays open and highlighted. Two actions: "Leave anyway" (discards draft, navigates) and "Stay & finish" (dismisses warning, returns focus to editor). The clicked nav button gets a subtle yellow border to show it triggered the warning.
  12. [12] Past entry banner — RESOLVED — Past Entry View replaces the breadcrumb with a full-width muted banner stating the entry date and "Read-only" (Q3 resolved: visually distinct, not just a banner on the same design). The entire screen background, header, and verse text are desaturated/muted. No "+" icons appear anywhere. URL is the entry route, not the content route.
  13. [13] Annotations in past entry view — Annotations are the primary content in this view — they're what the user came to read. They render the same structurally (left border bar, sans-serif, timestamp) but in a muted palette consistent with the read-only mode. No "Add another note" link.

  • ✅ Save error preserves note text and offers retry
  • ✅ Loading skeleton shown
  • ✅ Navigate-away warning is inline (not a modal) — doesn't block the page
